Cairn Architecture Note 对比通用 Agent + MCP / Skills
State-Space Search vs Tool-Augmented Agent

Cairn 不是另一个 Agent，而是让多个 Agent 协同搜索的运行架构。

Claude Code 这类通用 Agent 搭配 MCP / Skills，重点是增强“单个智能体会做什么”。 Cairn 的重点是管理“多个智能体如何围绕同一个目标积累事实、声明方向、并发探索、收敛到完成”。

Cairn

共享黑板 + 事实图 + 调度器
Origin / Goal问题被建模为起点到终点的搜索。
→
Facts / Intents / Hints所有 Agent 通过图间接协作，不直接聊天。
→
Bootstrap / Reason / ExploreDispatcher 统一派发任务并写回协议。

Claude Code + MCP / Skills

通用 Agent + 工具连接 + 专项能力
User Task用户给一个目标或问题。
→
Agent Reasoning模型自主规划、调用工具、编辑文件或查询资源。
→
MCP / Skills扩展能访问什么、会使用哪些流程和工具。

核心定位

Cairn 解决的是协作搜索的状态管理问题；MCP 和 Skills 解决的是工具与工作流增强问题。它们不是同一层级的东西。

Cairn：任务编排与状态空间搜索层

Cairn 把未知路径问题拆成可审计的图：Fact 是确认事实，Intent 是探索方向，Hint 是外部判断。 多个 worker 只围绕这张共享图行动，Dispatcher 负责认领、心跳、超时、本机进程、写回和收敛。

多 Agent 并发 事实图审计 任务租约 工作区隔离

Claude Code + MCP / Skills：单 Agent 能力扩展层

通用 Agent 通过 MCP 连接外部系统，通过 Skills 获得特定领域的操作流程。 它更像一个能力很强的执行者：能读代码、调用工具、写文件、查资料，但长期共享状态通常依赖上下文、文件或外部系统。

工具调用 本地代码修改 领域技能 交互式执行

能力矩阵

真正的区别在于“谁负责记忆状态、谁负责协作、谁负责收敛”。

维度
Cairn
Claude Code + MCP / Skills
目标层级
解决“未知路径搜索”从 origin 到 goal，不预设固定流程。
解决“当前任务执行”围绕用户当前指令进行规划、调用工具、产出结果。
状态管理
显式状态Facts、Intents、Hints 持久化在服务端，可回放、可审计。
隐式或外部状态依赖对话上下文、文件、工具返回、人工记录或外部项目系统。
协作方式
Stigmergy 黑板协作Agent 不直接沟通，通过共享图互相影响。
单 Agent 主导MCP / Skills 增强同一个 Agent 的能力；多 Agent 协作需额外系统设计。
工具关系
工具是 worker 的执行环境Dispatcher 只要求 worker 返回结构化结果。
工具是 Agent 的能力边界MCP 决定能连接哪些系统，Skills 决定擅长哪些流程。
失败处理
协议级恢复心跳超时释放 intent，任务可重派，失败不会污染事实图。
会话级恢复通常由 Agent 重新尝试、用户介入或工具自身报错处理。
典型场景
授权渗透测试、CTF、漏洞研究、复杂探索适合目标明确但路径未知、需要并发试探的问题。
编码、调试、文档、运维、检索、自动化适合明确任务和可直接调用工具完成的工作。

Cairn 的运行方式

它把通用 Agent 放进可控调度框架里，让 Agent 只做一件事：读任务、探索、返回结构化事实。

建模

用户创建 Project，写入 origin 和 goal。问题从一开始就是可追踪的图。

派发

Dispatcher 根据图状态选择 bootstrap、explore 或 reason。

执行

Claude Code、Codex、Pi 等 worker 在项目工作区内执行具体探索。

写回

Dispatcher 校验 JSON 输出，把结果写成 Fact、Intent 或 Complete。

收敛

图增长后继续触发新探索，直到某些事实共同满足 goal。

一句话结论

MCP / Skills 让 Agent 更会用工具；Cairn 让一组 Agent 围绕同一个目标有组织地搜索、记账、并发和收敛。

Cairn 可调度 Claude Code 不是替代关系
用 Cairn 当任务像“从未知攻击面拿到 flag / shell / 证明”这样路径开放、需要多轮探索、并且希望保留完整因果链。
用 Claude Code + MCP / Skills 当任务更像“修一个 bug、查一个接口、生成一个报告、调用某个系统完成操作”，目标清楚且主要依赖工具能力。
组合使用 把 Claude Code 当作 Cairn 的 worker，再让它通过 MCP / Skills 获得更强工具能力；Cairn 负责全局调度和事实图。

# Web SSRF

适用于 Web SSRF / 伪协议 / 重定向绕过类题目的容器内技能。

## 目标
- 先确认注入点是否真的发起请求
- 区分直连、跟随跳转、协议受限、回显受限
- 在拿不到完整回显时，优先收集 side effect 和错误模式

## 工作方式
1. 先用最小请求确认参数行为，不要一开始就堆复杂 payload。
2. 先记录目标对这些输入的差异：
   - `http://example.com`
   - `http://127.0.0.1`
   - `file:///etc/passwd`
   - `gopher://` / `redis://`
3. 如果响应不稳定、超时或站点下线，优先把它作为 blocker 事实记录下来，不要重复拆出同义探索路线。
4. 如果题目提示涉及 302，单独验证“是否跟随跳转”与“跳转后的主机/IP 校验位置”。
5. 如需 bash/curl，请优先输出：
   - 命令目的
   - 关键响应头
   - 是否出现 flag 线索或本地服务特征

## 输出偏好
- 用简短结论总结“这一步验证了什么”
- 不重复生成与已有事实等价的意图
- 如果当前方向已被阻断，明确说明阻断原因和下一条更高价值路线
